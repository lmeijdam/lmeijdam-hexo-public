Deploy Angular App to Azure

2017_09
Angular

A while ago I started blogging and I noticed I let it fade away a little bit. I was traveling for an amount of 6 months in New Zealand and had a lot to think about. On blogging I was brainstorming about a new post and thought of combining some new, great sounding, technologies which I tried out somewhat. Although I was busy creating a project on my own in the last days, I was struggling with hosting an Angular (v2+) Application on Azure App Services.

Prerequisites

Before we head into coding, some tools are needed to get along this how-to. The tools are not mandatory as you can switch some tools with some of your own preference. In a nutshell, make sure you have the following installed;

  • NodeJs (https://nodejs.org/en/)
  • GIT (https://git-scm.com/)
  • Visual Studio Code (VSCode, https://code.visualstudio.com/)
  • Azure CLI (Azure CLI)
  • Angular CLI (Angular CLI)

First time using Azure? No problem… you can try it for free for 30 days! Find more information on https://azure.microsoft.com.

Before deploying the App to Azure we need to make sure it’s getting built. One strong key of building the application is the overall bundling and minifying of the files. Although the application is being built with ‘ng serve’ it’s better to build it for ‘production’ as this would enable tree shaking and Ahead-of-time compilation. The overall files are smaller in size and helps increasing loading and performance.
